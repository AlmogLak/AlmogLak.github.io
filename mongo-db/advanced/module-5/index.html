Replica-sets

  • Group of mongod processes that maintain the same data set
  • Provide redundancy and high availability
  • Provide increased read capacity
  • Automatic failover
  • Client writes to a primary Mongo DB server which sends replicas to the secondary's

Sharding

  • Method to distribute data across multiple machines
  • Support deployments with large data sets
  • Sharding types: Range, Hash and Tag-Aware
  • DB operations are transparently routed across the cluster through a routing process proxy
  • There is a config server which stores metadata and configuration settings for cluster
